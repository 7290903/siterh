# Инструкция по деплою на GitHub Pages

## Автоматический деплой через GitHub Actions (Рекомендуется)

Проект настроен для автоматического деплоя при каждом push в ветку `main`.

### Настройка GitHub Pages

1. Перейдите в настройки репозитория на GitHub
2. Откройте раздел **Pages** (Settings → Pages)
3. В разделе **Source** выберите:
   - **Source**: `GitHub Actions`
4. Сохраните изменения

### Как это работает

- При каждом push в ветку `main` автоматически запускается workflow
- Проект собирается и деплоится на GitHub Pages
- Сайт будет доступен по адресу: `https://7290903.github.io/siterh/`

## Ручной деплой

### Вариант 1: Через скрипт (Windows)

Запустите файл `deploy.bat` двойным кликом. Скрипт:
- Установит зависимости (если нужно)
- Соберет проект
- Отправит на GitHub Pages

### Вариант 2: Через команды

```bash
# Установить зависимости (если еще не установлены)
npm install

# Установить gh-pages (если еще не установлен)
npm install --save-dev gh-pages

# Собрать и задеплоить
npm run deploy
```

## Проверка деплоя

После деплоя сайт будет доступен по адресу:
**https://7290903.github.io/siterh/**

Изменения могут появиться через 1-5 минут после деплоя.

## Важные замечания

1. **Base path**: Проект настроен на использование base path `/siterh/` для GitHub Pages
2. **Пути к ресурсам**: Все пути к видео и изображениям используют относительные пути через `import.meta.env.BASE_URL`
3. **Ветка**: Деплой происходит из ветки `main` в ветку `gh-pages` (автоматически)

## Устранение проблем

### Ошибка при деплое
- Проверьте, что GitHub Pages включен в настройках репозитория
- Убедитесь, что у вас есть права на запись в репозиторий
- Проверьте логи в разделе Actions на GitHub

### Ресурсы не загружаются
- Убедитесь, что base path настроен правильно в `vite.config.ts`
- Проверьте, что все пути используют `import.meta.env.BASE_URL`

### Сайт показывает 404
- Убедитесь, что GitHub Pages настроен на использование GitHub Actions
- Проверьте, что workflow успешно выполнился в разделе Actions
