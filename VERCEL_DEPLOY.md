# Инструкция по деплою на Vercel

## Автоматическая настройка

Проект настроен для автоматической работы на Vercel. Конфигурация определяет платформу деплоя и использует правильный base path:
- **Vercel**: base path = `/` (корневой путь)
- **GitHub Pages**: base path = `/siterh/`

## Деплой на Vercel

### Способ 1: Через веб-интерфейс Vercel

1. Перейдите на https://vercel.com
2. Войдите в аккаунт (через GitHub)
3. Нажмите "Add New Project"
4. Импортируйте репозиторий `7290903/siterh`
5. Настройки проекта:
   - **Framework Preset**: Vite
   - **Root Directory**: `./` (корень проекта)
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
6. Нажмите "Deploy"

### Способ 2: Через Vercel CLI

```bash
# Установите Vercel CLI (если еще не установлен)
npm i -g vercel

# Войдите в аккаунт
vercel login

# Задеплойте проект
vercel

# Для production деплоя
vercel --prod
```

## Важные замечания

1. **Base path**: На Vercel автоматически используется корневой путь `/`
2. **Переменные окружения**: Vercel автоматически устанавливает `VERCEL=1`, что определяет платформу
3. **Роутинг**: Файл `vercel.json` настроен для правильной обработки SPA роутинга

## Проверка деплоя

После деплоя сайт будет доступен по адресу, который предоставит Vercel (например: `https://siterh.vercel.app`).

Проверьте:
- ✅ Сайт загружается без ошибок
- ✅ JS и CSS файлы загружаются правильно
- ✅ Нет ошибок 404 для ресурсов

## Устранение проблем

### Ошибки 404 для JS/CSS файлов

Если видите ошибки 404 для `index-xxx.js` или `index-xxx.css`:
1. Убедитесь, что в настройках Vercel:
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
2. Проверьте логи сборки в Vercel Dashboard
3. Убедитесь, что переменная `VERCEL` установлена (устанавливается автоматически)

### Проблемы с путями

Если ресурсы не загружаются:
- Проверьте, что base path = `/` на Vercel
- Убедитесь, что в логах сборки видно: "Platform: Vercel"

## Одновременный деплой на GitHub Pages и Vercel

Проект настроен для работы на обеих платформах:
- **GitHub Pages**: автоматический деплой через GitHub Actions с base path `/siterh/`
- **Vercel**: автоматический деплой с base path `/`
