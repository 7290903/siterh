# Устранение проблем с деплоем на GitHub Pages

## Проблема: Ошибки 404 для ресурсов

Если вы видите ошибки типа:
- `GET https://7290903.github.io/src/main.tsx 404`
- `GET https://7290903.github.io/vite.svg 404`

### Решение 1: Очистка кеша браузера

1. **Жесткая перезагрузка страницы:**
   - Windows/Linux: `Ctrl + Shift + R` или `Ctrl + F5`
   - Mac: `Cmd + Shift + R`

2. **Очистка кеша браузера:**
   - Откройте DevTools (F12)
   - Правый клик на кнопке обновления → "Очистить кеш и жесткая перезагрузка"

3. **Режим инкогнито:**
   - Откройте сайт в режиме инкогнито/приватном режиме
   - Это исключит влияние кеша

### Решение 2: Проверка деплоя

1. Убедитесь, что последний workflow завершился успешно:
   - Перейдите в https://github.com/7290903/siterh/actions
   - Проверьте, что последний деплой завершился без ошибок

2. Подождите 2-3 минуты после деплоя:
   - GitHub Pages может обновляться с задержкой

3. Проверьте правильный URL:
   - Должен быть: `https://7290903.github.io/siterh/`
   - НЕ: `https://7290903.github.io/` (без `/siterh/`)

### Решение 3: Перезапуск деплоя

Если проблема сохраняется:

1. **Через GitHub Actions:**
   - Откройте https://github.com/7290903/siterh/actions
   - Выберите "Deploy to GitHub Pages"
   - Нажмите "Run workflow"

2. **Через скрипт:**
   - Запустите `redeploy.bat`

3. **Вручную:**
   ```bash
   git commit --allow-empty -m "Перезапуск деплоя"
   git push origin main
   ```

### Решение 4: Проверка собранных файлов

Убедитесь, что в собранном `dist/index.html` пути правильные:

```bash
npm run build
cat dist/index.html
```

Должны быть пути вида `/siterh/assets/...`, а не `/src/main.tsx`

## Проверка работоспособности

После деплоя проверьте:

1. ✅ Workflow завершился успешно
2. ✅ В логах видно "✓ Base path found in index.html"
3. ✅ Открыт правильный URL с `/siterh/`
4. ✅ Кеш браузера очищен
5. ✅ Прошло 2-3 минуты после деплоя

## Контакты

Если проблема не решается, проверьте:
- Логи в GitHub Actions
- Консоль браузера (F12)
- Network tab в DevTools
